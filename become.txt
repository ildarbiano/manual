
Privilege Escalation Options:	control how and which user you become as on target hosts
Опции повышения привилегий:	контролируйте, как и каким пользователем вы становитесь на целевых хостах
 -b, --become          run operations with become (does not imply password prompting) запуск операций с помощью become (не подразумевает запроса пароля)


ansible_become
переопределяет become директиву и решает, использовать ли повышение привилегий или нет.

become_user
установить для пользователя с желаемыми привилегиями — пользователь, которым вы становитесь , а НЕ пользователь, под которым вы входите в систему. НЕ подразумевает , чтобы разрешить установку на уровне хоста. become:true . Значение по умолчанию — root



[defaults]
ansible_become_user	=true
ansible_become_password	=./
ansible_become_pass	=
Установите пароль повышения привилегий. Подробнее о том, как избежать секретов в открытом виде, смотрите в разделе Использование зашифрованных переменных и файлов
=======
установить пользователя, которым вы становитесь посредством повышения привилегий; не подразумевает ansible_become: true

Например, если вы хотите запустить все задачи root на сервере с именем web@server, но можете подключиться только как manager пользователь, вы можете использовать следующую запись инвентаря:

web@server ansible_user=manager ansible_become=true
=======

vault_password_file  

Файл пароля хранилища для использования. Эквивалентно --vault-password-file или --vault-id. 
Если он исполняемый, он будет запущен, а полученный stdout будет использован в качестве пароля
